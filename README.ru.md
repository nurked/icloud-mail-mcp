# imail-mcp

> 🇬🇧 **Documentation in English:** [README.md](README.md)

[MCP](https://modelcontextprotocol.io)-сервер, который подключает Claude
(Cowork / Claude Code / Claude Desktop) к вашей почте, чтобы он мог
**читать, искать, отправлять и сортировать** письма — прямо из диалога.

Первый поддерживаемый провайдер — **iCloud Mail**. Внутри всё построено на
провайдеро-независимом интерфейсе, поэтому адаптеры для Gmail / Microsoft 365
можно добавить позже, не трогая ни один инструмент.

---

## Возможности

- 📥 **Чтение и поиск** — список папок, поиск по отправителю / теме / тексту /
  дате, чтение целых писем и веток переписки, скачивание вложений.
- ✉️ **Отправка и ответы** — составление письма, ответ, ответ всем и пересылка
  с автоматически проставленными заголовками для корректной ветки переписки.
- 🗂️ **Сортировка** — перемещение, архивирование, удаление, пометка писем
  прочитанными/непрочитанными и флажками.
- 🔒 **Учётные данные хранятся локально** — в **связке ключей macOS (Keychain)**,
  а не в текстовом файле или переменной окружения.
- 🛑 **Ничего не отправляется молча** — Claude обязан сначала показать
  предпросмотр письма и получить ваше явное согласие, прежде чем что-либо уйдёт
  из вашего ящика.

## Почему требуется одноразовая настройка (и нет кнопки «Войти через Apple»)

Apple **не предоставляет** OAuth-/consent-поток для стороннего доступа к
iCloud Mail. Единственный поддерживаемый способ — стандартные **IMAP + SMTP** с
**паролем для приложения** (он обязателен, потому что у всех аккаунтов iCloud
включена двухфакторная аутентификация).

Поэтому настройка — это один разовый шаг: сгенерировать такой пароль один раз,
передать его серверу — и всё. Он проверяется **живым входом по IMAP** и затем
сохраняется в **связке ключей macOS** — не на диске и не в профиле оболочки.

> **Замечание о платформе:** хранение учётных данных использует встроенную
> утилиту macOS `security`, поэтому сейчас сервер рассчитан на **macOS**.
> (Для поддержки Linux/Windows достаточно заменить бэкенд хранилища.)

## Установка и подключение

Требуется **Node.js ≥ 18**.

```bash
git clone git@github.com:nurked/icloud-mail-mcp.git
cd icloud-mail-mcp
npm install
npm run build
npm run setup          # или: node dist/bin/setup.js
```

`setup` открывает локальную страницу по адресу **http://127.0.0.1:4577**
(привязана только к localhost — наружу машины она не выходит). Эта страница:

1. Даёт ссылку на страницу Apple для создания пароля приложения.
2. Принимает ваш адрес iCloud и 16-значный сгенерированный пароль.
3. Делает **живой тестовый вход по IMAP** для проверки.
4. Сохраняет учётные данные в **связку ключей**.

### Как создать пароль для приложения

1. Откройте [appleid.apple.com](https://appleid.apple.com/account/manage) →
   **Вход и безопасность → Пароли приложений**.
2. Нажмите **Создать пароль для приложения**, назовите его `imail-mcp`.
3. Скопируйте 16-значный пароль и вставьте его на странице настройки.

## Подключение к Claude

Добавьте сервер в конфигурацию MCP (Claude Desktop / Cowork / Claude Code,
секция `mcpServers`):

```json
{
  "mcpServers": {
    "imail": {
      "command": "node",
      "args": ["/абсолютный/путь/к/icloud-mail-mcp/dist/index.js"]
    }
  }
}
```

> Если установить пакет глобально (`npm link` или `npm i -g`), можно вместо
> этого указать `"command": "imail-mcp"` без `args`.

Перезапустите Claude — и инструменты `imail` станут доступны.

## Инструменты

| Инструмент | Что делает |
| --- | --- |
| `list_mailboxes` | Список папок с их IMAP-путями и специальными ролями |
| `search_messages` | Поиск по from / to / subject / text / дате / непрочитанным (фильтры объединяются по И) |
| `get_message` | Полные заголовки, текст + HTML-тело, метаданные вложений |
| `get_thread` | Вся переписка, от старых к новым |
| `download_attachment` | Сохранить вложение во временный файл и вернуть путь |
| `compose_preview` | Составить письмо для проверки — **не отправляет** |
| `send_email` | Отправить сейчас (только после вашего согласия) |
| `save_draft` | Положить в «Черновики», чтобы вы отправили сами |
| `reply` | Ответить / ответить всем, заголовки ветки берутся из оригинала |
| `forward` | Переслать новым получателям с цитируемым заголовком |
| `move_message` | Переместить письмо в другую папку |
| `set_flags` | Пометить прочитанным/непрочитанным, поставить/снять флажок |
| `archive_message` | Переместить в «Архив» |
| `delete_message` | Переместить в «Корзину» (удалить окончательно, если уже там) |

### Отправка никогда не происходит молча

Сервер передаёт Claude инструкцию **всегда** показывать вам `compose_preview` и
получать явное согласие перед вызовом `send_email` (или `reply` / `forward`).
Если вы предпочитаете отправить письмо сами — попросите Claude использовать
`save_draft`, и письмо окажется в папке «Черновики».

## Как это устроено

```
Claude  ──stdio──▶  imail-mcp  ──IMAP (imap.mail.me.com:993)──▶  iCloud
                       │        └─SMTP (smtp.mail.me.com:587)──▶  iCloud
                       └─ учётные данные ◀── связка ключей macOS
```

- **IMAP** ([imapflow](https://www.npmjs.com/package/imapflow)) отвечает за
  чтение и сортировку; **SMTP** ([nodemailer](https://nodemailer.com)) — за
  отправку; тела писем разбираются через
  [mailparser](https://www.npmjs.com/package/mailparser).
- Провайдер подключается **лениво**, при первом вызове инструмента, поэтому само
  добавление сервера в Claude не трогает ваш ящик, пока вы им не воспользуетесь.
- Транспорт MCP `stdio` владеет stdout, поэтому всё логирование идёт в stderr,
  чтобы не засорять поток протокола.

### Структура проекта

```
src/
  index.ts              MCP-сервер и определения инструментов
  bin/setup.ts          точка входа `imail-mcp-setup`
  auth/
    setup-server.ts     локальная страница подключения + живая проверка IMAP
    keychain.ts         хранение в связке ключей macOS (через утилиту `security`)
  providers/
    types.ts            провайдеро-независимый контракт MailProvider
    icloud.ts           реализация iCloud по IMAP/SMTP
```

## Добавление нового провайдера

Инструменты работают только с интерфейсом `MailProvider`, поэтому новый бэкенд —
это один новый файл:

1. Реализуйте `MailProvider` (см.
   [`src/providers/types.ts`](src/providers/types.ts)) в новом файле в
   `src/providers/`.
2. Выберите его в [`src/index.ts`](src/index.ts).

Инструменты остаются без изменений.

## Решение проблем

- **«No iCloud credentials found»** — сначала выполните `npm run setup`; сервер
  читает данные из связки ключей при первом вызове инструмента.
- **«Login failed» при настройке** — убедитесь, что вы использовали **пароль для
  приложения**, а не обычный пароль Apple ID.
- **Переподключение / смена аккаунта** — просто запустите `npm run setup` снова;
  он перезапишет сохранённые данные (обновление `-U`) после новой проверки.
